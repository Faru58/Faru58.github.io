内部类和分部类

内部类

基本概念：
在一个类中再声明一个类

特点：
使用时要用包裹者点出自己

作用：
亲密关系的表现

注意：
访问修饰符作用很大

class Person
 {
     public int age;
     public string name;
     public Body body;

     public class Body
     {
         Arm leftArm;
         Arm rightArm;
         //class在内部类里默认private
         class Arm
         {

         }
     }

     
 }


 internal class Program
 {
     static void Main(string[] args)
     {
         Person p = new Person();
         Person.Body body = new Person.Body();
         //Person.Body.Arm arm = new Person.Body.Arm(); 会报错
     }
 }

分部类

基本概念：
把一个类分成几部分声明

关键字 partial

作用：
1.分布描述一个类
2.增加程序的拓展性

注意：
1.分部类可以写在多个脚本文件中
2.分部类的访问修饰符要一致
3.分部类中不能有重复成员

partial class Student
{
    public bool sex;
    public string name;

}

//一个类 分开写
partial class Student
{
    public int number;
    public void Speak(string str)
    {

    }

}

internal class Program
{
    static void Main(string[] args)
    {
        Student student = new Student();
        student.Speak(string.Empty);
    }
}

分部方法

基本概念：
将方法的声明和实现分离

特点：
1.不能加访问修饰符 默认private
2.只能在分部类中声明
3.返回值只能是void
4.可以有参数 不能用out关键字

//声明和实现分到两个分部类里面
    partial class Student
    {
        public bool sex;
        public string name;
        //声明
        partial void Speak();
    }

    //一个类 分开写
    partial class Student
    {
        public int number;
        //实现
        partial void Speak()
        {
            //实现逻辑
        }
    }
