继承中的构造函数

继承中的构造函数

继承中的构造函数

基本概念

特点：
当声明一个子类对象时
先执行父类的构造函数
再执行子类的构造函数

注意：
1.父类的无参构造 很重要
2.子类可以通过base关键字 代表父类 调用父类构造

继承中构造函数的执行顺序

1.父类的父类的构造函数
2.父类的构造函数
3.子类的构造函数
即从高到低依次调用

class GameObject
{
    public GameObject()
    {
        Console.WriteLine("GameObject的构造函数");
    }
}

class Player : GameObject
{
    public Player()
    {
        Console.WriteLine("Player的构造函数");
    }
}

class MainPlayer : Player
{
    public MainPlayer()
    {
        Console.WriteLine("MainPlayer的构造函数");
    }
}

internal class Program
{
    static void Main(string[] args)
    {
        MainPlayer mp = new MainPlayer();
    }
}

父类的无参构造函数很重要

重点：
无参构造函数会被有参构造函数顶替

class Father
{
    public Father(int i)
    {
        Console.WriteLine("Father构造函数");
    }
}

class Son : Father
{
    //默认调用父类的无参构造函数，但父类中无参构造函数被顶替，所以报错
}

internal class Program
{
    static void Main(string[] args)
    {
        Son s = new Son();
    }
}

通过base调用制定父类构造

class Father
{
    public Father(int i)
    {
        Console.WriteLine("Father构造函数");
    }
}

class Son : Father
{
    public Son(int i) : base(i)
    {
        Console.WriteLine("Son一个参数的构造函数");
    }
    public Son(int i, string str) : this (i)
    //this其实是先调用base(i),是一种间接的方式;如果用base会直接执行父类构造函数
    {
        Console.WriteLine("Son两个参数的构造函数");
    }
}

internal class Program
{
    static void Main(string[] args)
    {
        Son s1 = new Son(1, "123");


    }
}

注意：
this和base的区别
this:自己的另一个对应构造函数
base:父类的对应构造函数

练习题

using System;

namespace Lesson14_继承中的构造函数
{
    enum E_TypeOfWork
    {
        Programmer,
        Planner,
        Artist,
    }
    enum E_WorkContent
    {
        program,
        plan,
        paint,
    }
    class Worker
    {
        public E_TypeOfWork typeOfWork;
        public E_WorkContent workContent;
        public Worker(E_TypeOfWork typeOfWork, E_WorkContent workContent)
        {
            Console.WriteLine("我是打工人");
        }
    }
    
    class Programmer : Worker
    {
        public Programmer(E_TypeOfWork typeOfWork, E_WorkContent workContent) : base(typeOfWork, workContent)
        {
            Console.WriteLine("{0} is {1}ing", typeOfWork, workContent);
        }
    }
    class Planner : Worker
    {
        public Planner(E_TypeOfWork typeOfWork, E_WorkContent workContent) : base(typeOfWork, workContent)
        {
            Console.WriteLine("{0} is {1}ing", typeOfWork, workContent);
        }
    }
    class Artist : Worker
    {
        public Artist(E_TypeOfWork typeOfWork, E_WorkContent workContent) : base(typeOfWork, workContent)
        {
            Console.WriteLine("{0} is {1}ing", typeOfWork, workContent);
        }
    }
    internal class Program
    {
        static void Main(string[] args)
        {
            Programmer programmer = new Programmer(E_TypeOfWork.Programmer, E_WorkContent.program); 
            Planner planner = new Planner(E_TypeOfWork.Planner, E_WorkContent.plan);
            Artist artist = new Artist(E_TypeOfWork.Artist, E_WorkContent.paint);
        }
    }
}
