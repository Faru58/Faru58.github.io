静态成员

静态成员

基本概念

基本关键字： static
用static修饰的成员变量、方法、属性等的成为静态成员

特点：直接用类名点出使用
Console就是一个静态类

声明静态成员变量 及 使用

class Test
{
    //public和static 前后无所谓
    public static float PI = 3.1415926f;
    //成员变量
    public int testInt = 100;

    //静态成员方法
    public static float CalcCricle(float r)
    {
        return PI * r * r;
    }
    //普通成员方法
    public void TestFun()
    {
        Console.WriteLine("123");
    }
}
internal class Program
{
    static void Main(string[] args)
    {
        //类名.静态成员变量  不需要实例化
        Console.WriteLine(Test.PI);
        Console.WriteLine(Test.CalcCricle(2));
    }
}

为什么可以直接点出来使用

程序中是不能无中生有的
需要使用的对象，变量，函数都是要在内存中分配内存空间的
之所以要实例化对象，目的就是分配内存空间，在程序中产生一个抽象的对象

静态成员的特点
程序开始运行时，就会分配内存空间，得以直接使用
静态成员和程序是同生共死的
只要使用了它，知道程序结束时才会被释放
所以一个静态成员就会有自己唯一的一个“内存小房间”
这让静态成员就有了唯一性
在任何地方使用都是用的小房间里的内容，改变了它也是改变小房间里的内容

静态函数非静态成员/非静态函数静态成员

成员变量只能将对象实例化出来后，才能点出来使用，不能无中生有
不能直接使用非静态函数，否则会报错
静态函数中不可以直接使用非静态成员
非静态函数中可以直接使用静态成员

静态成员对于我们的作用

静态变量：
1.常用唯一变量的声明
2.方便别人获取的对象声明

静态方法：
常用的唯一的方法声明
比如：相同规则的数学计算相关函数

常量和静态变量

const 可以理解为特殊的 static
相同点：
他们都可以通过类名.点出使用
不同点：
1.const必须初始化，不能修改; static没有这个规则
2.const只能修饰变量; static可以修饰很多(成员方法等)
3.const一定是写在访问修饰符后面的(3P后面); static没有这个要求
