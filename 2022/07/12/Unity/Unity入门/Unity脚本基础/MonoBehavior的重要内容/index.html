MonoBehavior的重要内容

MonoBehavior的重要内容

MonoBehavior

成员

this代表脚本依附的对象

  1. 获取依附的gameObject的名字

    print(this.gameObject.name);

  2. 获取依附的gameObject的位置信息

    print(this.transform.position); //位置
print(this.transform.eulerAngles); //角度
print(this.transform.lossyScale); //缩放大小

  3. 获取脚本是否激活

    this.enabled = false; //失活

  4. 获取别的脚本依附的gameObject的信息

    public Lesson3 otherLesson3;

print(otherLesson3.gameObject.name); 
print(otherLesson3.transform.position);

方法

得到依附对象上挂载的其他脚本

  1. 得到自己挂载的单个脚本

根据脚本名获取(如果获取失败，即没有对应的脚本，会默认返回空)

Lesson3_Test t = this.GetComponent("Lesson3_Test") as Lesson3_Test;
print(t);

根据Type获取

t = this.GetComponent(typeof(Lesson3_Test)) as Lesson3_Test;
print(t);

根据泛型获取(建议使用泛型获取，因为不用二次转换)

t = this.GetComponent<Lesson3_Test>();
print(t);

  1. 得到自己挂载的多个脚本

    Lesson3[] array = this.GetComponents<Lesson3>();
print(array);

List<Lesson3> list = new List<Lesson3>();
this.GetComponents<Lesson3>(list);
print(list.Count);

  2. 得到子对象挂载的脚本(它默认也会找自己身上是否挂载脚本)
    函数是有一个参数的，默认不传是FALSE，意思就是如果子对象失活，是不会去找这个对象上是否有某个脚本的；如果传TRUE，即使失活也会找

    t = this.GetComponentInChildren<Lesson3_Test>();
print(t);

List<Lesson3> list2 = new List<Lesson3>();
this.GetComponentsInChildren<Lesson3>(list);
print(list2.Count);

  3. 得到父对象挂载的脚本(它默认也会找自己身上是否挂载脚本)

    t = this.GetComponentInParent<Lesson3_Test>();
print(t);

  4. 尝试获取脚本
    提供了一种更加安全的 获取单个脚本的方法 如果得到了会返回TRUE
    然后再逻辑处理即可

    Lesson3_Test l3t;
if(this.TryComponent<Lesson3_Test>(out l3t)){

}

this.transform代表脚本对象依附的gameObject的游戏对象的位置相关信息
