串

串

实际上是向量
由来自字母表∑的字符所组成的有限序列 S = a0,a1,…,a(n-1)
通常字符的种类不多，而串长 = n >> |∑| (字母表长度)

术语

  1. 相等： S[0, n) = T[0, m)
    长度相等(n = m)，且对应的字符均相同(S[i] = T[i])

  2. 子串： S.substr(i, k) = S[i, i+k)
    亦即从S[i]起连续k个字符

  3. 前缀： S.prefix(k) = S.substr(0, k) = S[0, k)
    亦即S中最靠前的k个字符

  4. 后缀： S.suffix(k) = S.substr(n-k, n) = S[n-k, n)
    亦即S中最靠后的k个字符

  5. 联系： S.substr(i, k) = S.prefix(i+k).suffix(k)

  6. 空串： S[0, n=0) 是任何串的子串，前缀，后缀

蛮力匹配

自左向右，以字符为单位，依次移动模式串，直到在某个位置发现匹配
|∑| 越小，最坏情况的出现概率越高
m 越大，最坏情况的后果更加严重
对于长度为n的文本串，长度为m的模式串，时间复杂度为O(m*n)

KMP算法

蛮力匹配低效原因：此前比对过的字符，将再次参与比对

不变性： T[i-j, i) = P[0, j)， 即已掌握T[i-j, i)的全部信息
记忆性： 失败后的下一轮，不必再次比对T[i-j, i)，直接从T[i]开始往后比对
预知力： 失败后，将模式串中前一个字符与刚才失败的那个字符重新对齐

为此，需要事先确定t，即构建查询表next[0, m)

next[]表

自匹配：

  1. 快速右移
    集合N(P,j) = P[0, t) == P[j-t, j), 即在P[j]的前缀P[0, j)中，所有匹配真前缀和真后缀的长度
    一旦失配(T[i] != P[j]),可以从N(P,j)中取t，继续比对

  2. 避免回退
    通配哨兵 next[0] = -1 (默认比对必然成功)

构造next[]表
next[j] = 在P[0, j)中，最大自匹配的真前缀和真后缀的长度
next[j+1] <= next[j] + 1，当且仅当P[j] == P[ next[j] ]时取等号

由此，我们需要充分利用每一次对比的信息，可提高算法效率
分摊分析 时间复杂度为O(n+m)
