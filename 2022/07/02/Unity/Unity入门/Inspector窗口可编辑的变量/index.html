Inspector窗口可编辑的变量

Inspector窗口可编辑的变量

Inspector窗口可编辑的变量

Inspector窗口可编辑的变量就是脚本中的成员变量

私有和保护的变量无法在窗口中显示编辑

让私有和保护的变量可以在窗口中显示编辑

加上强制序列化字段特性 [SerializeField]
所谓序列化就是把一个对象保存到一个文件或数据库字段中去

[SerializeField]
private int privateInt;
[SerializeField]
protected string protectedStr;

公共的变量可以在窗口中显示编辑

让公共的变量无法在窗口中显示编辑

加上序列化特性 [HideInInspector]

[HideInInspector]
public int publicInt2;

大部分类型都可以在窗口中显示编辑

字典、自定义类型/结构体 不能显示编辑

让自定义类型可以在窗口中显示编辑

加上序列化特性 [System.Serializable]

但字典怎么都不行

一些辅助特性

  1. 分组说明特性 Header
    为成员变量分组

    [Header("分组说明")]

  2. 悬停注释 Tooltip
    为成员变量添加说明

    [Tooltip("悬停说明")]

  3. 间隔特性 Space()
    让两个字段间出现间隔

    [Space()]

  4. 修饰数值的滑条范围 Range

    [Range(最小值，最大值)]

  5. 多行显示字符串 Multiline
    默认不写参数显示3行
    写参数就是对应行

    [Multiline(行数)]

  6. 滚动条显示字符串 TextArea
    默认不写参数为超过3行显示滚动条

    [TextArea(3, 4)] //最少显示3行，超过4行显示滚动条

  7. 为变量添加快捷方法 ContextMeunItem
    参数1：显示按钮名
    参数2：方法名，不能有参数(访问修饰符无所谓)

    [ContextMeunItem("显示按钮名", "方法名")] //最少显示3行，超过4行显示滚动条

  8. 为方法添加特性能够在Inspector中执行 ContextMeun
    在脚本那一栏右边的三个点里选

    [ContextMeun("测试方法名")]

注意

  1. Inspector窗口中的变量关联的就是对象的成员变量，运行时改变他们就是在改变成员变量
  2. 拖曳到GameObject对象后，再改变脚本中变量默认值，页面中不会改变
  3. 运行中修改的信息不会保存

练习题

题1

  1. 加上序列化特性 [HideInInspector]
  2. 加上强制序列化字段特性 [SerializeField]

题2

因为Unity中是通过反射得到类的信息，然后在Inspector窗口中显示字段信息，Unity内部通过反射获取字段的特性，当具有一些特殊特性时，便会做对应的处理
