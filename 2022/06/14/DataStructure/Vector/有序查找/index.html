二分查找

binSearch 反复确定中点
递归次数： O( log n )
查找长度： O( 1.50 * log n )

但还有改进空间：
转向左右分支前的关键码比较次数不等，而递归深度却相同
若能通过递归深度的不均衡，对转向成本的不均衡进行补偿，平均查找长度应能进一步缩短

Fibonacci查找

按黄金分割比切分
查找长度： O( 1.44 * log n )

直接改进二分查找

将中点的判断归入右侧分支
待查找区间缩短至1
[lo, mi), [mi, hi)

再次改进二分查找

区间宽度缩短至0
[lo, mi), (mi, hi)

插值查找

Interpolation Search

假设：已知有序算法中各元素随机分布的规律
则 [lo, hi) 内各元素应大致按照线性趋势增长
通过猜测轴点mi，可以极大地提高收敛速度

最坏情况 O(n)

总结

大规模：插值查找 （缩小规模，将问题规模缩小到足够小）
中规模：二分查找
小规模：顺序查找
