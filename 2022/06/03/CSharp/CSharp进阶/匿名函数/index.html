匿名函数

匿名函数

匿名函数

概念

没有名字的函数
匿名函数的使用主要是配合委托和事件进行使用
脱离委托和事件 是不会使用匿名函数的

语法

delegate ( 参数列表 )
{
    //函数逻辑
};

何时使用：
1.函数中传递委托参数时
2.委托或事件赋值时

使用

1.无参无返回

//这样声明匿名函数 只是在声明函数而已 还没有调用
Action a = delegate()
{
    Console.WriteLine("匿名函数逻辑");
};
//这里使用委托的时候 才是在调用
a();

2.有参

Action<int, string> b = delegate (int a, string b)
{
    Console.WriteLine(a);
    Console.WriteLine(b);
};

b(100, "頑張れ");

3.有返回值

Func<string> c = delegate()
{
    return "頑張れ";
};

Console.WriteLine(c());

4.一般情况会作为函数参数传递 或者 作为函数返回值

class Test
{
    public Action action;
    
    //作为参数传递时
    public void Do(int a, Action fun)
    {
        Console.WriteLine(a);
        fun();
    }

    //作为返回值
    public Action GetFun()
    {
        return delegate()
        {
            Console.WriteLine("函数内部返回的一个匿名函数的逻辑");
        };
    }
}

Test t = new Test();

//第一种 一步到位
t.Do(100, delegate()
{
    Console.WriteLine("随参数传入的匿名函数的逻辑");
});
//第二种 先把匿名函数用委托存起来
Action ac = delegate()
{
    Console.WriteLine("随参数传入的匿名函数的逻辑");
}
t.Do(50, ac);

//第一种 一步到位
t.GetFun()();
//第二种 先把匿名函数用委托存起来
Action ac2 = t.GetFun;
ac2();

匿名函数的缺点

添加到委托或事件容器中后 不记录 无法单独移除

练习题

using System;

namespace Lesson14_匿名函数
{
    internal class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine(Multiplication(10)(4));
        }

        static Func<int, int> Multiplication(int num1)
        {
            //这种写法会改变num1的生命周期
            return delegate(int num2)
            {
                return num1 * num2;
            };
        }
    }
}
